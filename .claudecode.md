# Instruções Globais — infra-lab-proxmox

Este projeto provisiona infraestrutura de laboratório em Proxmox: cluster Kubernetes e
stack CI/CD completa usando Terraform (provider `bpg/proxmox`) e Ansible.

## Estrutura do projeto

```
infra-lab-proxmox/
├── .claudecode.md                    ← este arquivo (instruções globais)
├── README.infraestructure.md         ← referência de acesso ao ambiente Proxmox
│
├── terraform-proxmox/                ← Stack: Cluster Kubernetes (master + 2 workers)
│   ├── main.tf / variables.tf / outputs.tf
│   └── terraform.tfvars.example
│
├── ansible-k8s/                      ← Stack: Cluster Kubernetes
│   ├── ansible.cfg / site.yml
│   ├── group_vars/all.yml            ← K8s 1.29, Calico, CIDRs
│   ├── inventory/ (tpl + script)
│   └── playbooks/ (01 a 05)
│
├── terraform-cicd/                   ← Stack: CI/CD (1 VM consolidada)
│   ├── main.tf / variables.tf / outputs.tf
│   └── terraform.tfvars.example
│
└── ansible-cicd/                     ← Stack: CI/CD (Gitea + Registry + Runner)
    ├── ansible.cfg / site.yml
    ├── group_vars/all.yml
    ├── inventory/ (tpl + script)
    ├── playbooks/ (01 a 04)
    └── templates/ (docker-compose.yml.j2, gitea-app.ini.j2, registry-config.yml.j2)
```

## Fluxo de execução completo

```
# Kubernetes
1. Preencher terraform-proxmox/terraform.tfvars com valores reais
2. cd terraform-proxmox && terraform init && terraform plan -out=tfplan.binary && terraform apply tfplan.binary
3. cd ../ansible-k8s && bash inventory/generate_inventory.sh
4. ansible-playbook -i inventory/hosts.yml site.yml

# CI/CD
1. Preencher terraform-cicd/terraform.tfvars com valores reais
2. cd terraform-cicd && terraform init && terraform plan -out=tfplan.binary && terraform apply tfplan.binary
3. cd ../ansible-cicd && bash inventory/generate_inventory.sh
4. ansible-playbook -i inventory/hosts.yml site.yml
5. (após criar token no Gitea UI) ansible-playbook ... --tags runner -e "gitea_runner_token=TOKEN"
```

## Regras deste projeto

- **Terraform**: nunca execute `apply` sem revisar o `plan` primeiro (`-out=tfplan.binary`)
- **Secrets**: credenciais do Proxmox via variáveis de ambiente (`source ~/.env.proxmox`), nunca em `.tfvars` versionados
- **Cloud-init**: IPs e chaves SSH são injetados exclusivamente via cloud-init — não altere manualmente as VMs
- **Inventário Ansible**: sempre regenere com `generate_inventory.sh` após `terraform apply`
- **Ansible**: execute `ansible-lint --profile production` antes de qualquer playbook em produção
- **Kubernetes**: versão `1.29` com containerd como CRI e Calico como CNI
- **CI/CD**: todos os serviços rodam como containers Docker com memory limits definidos

## Pré-requisitos

```bash
# Ferramentas necessárias
terraform >= 1.6.0
ansible >= 2.14
ansible-lint >= 6.0
python3 >= 3.10
jq >= 1.6

# Variáveis de ambiente (carregar antes de qualquer operação)
source ~/.env.proxmox

# Chave SSH para as VMs
ssh-keygen -t ed25519 -f ~/.ssh/id_ed25519_proxmox -C "lab-k8s-proxmox"
```
