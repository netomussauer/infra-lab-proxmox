# Instruções Globais — infra-lab-proxmox

Este projeto provisiona um cluster Kubernetes de laboratório em ambiente Proxmox local
usando Terraform (provider `bpg/proxmox`) e Ansible (kubeadm + containerd).

## Estrutura do projeto

```
infra-lab-proxmox/
├── .claudecode.md                  ← este arquivo (instruções globais)
├── README.infraestructure.md       ← referência de acesso ao ambiente Proxmox
├── terraform-proxmox/              ← provisionamento de VMs via Terraform
│   ├── main.tf                     ← provider + recursos de VM
│   ├── variables.tf                ← todas as variáveis de entrada
│   ├── outputs.tf                  ← outputs usados pelo inventário Ansible
│   └── terraform.tfvars.example   ← valores de exemplo (não versionar .tfvars real)
└── ansible-k8s/                    ← configuração do cluster Kubernetes
    ├── ansible.cfg                 ← configuração do Ansible
    ├── site.yml                    ← playbook principal (orquestra todos os demais)
    ├── group_vars/all.yml          ← variáveis globais
    ├── inventory/
    │   ├── hosts.yml.tpl           ← template do inventário
    │   └── generate_inventory.sh  ← gera hosts.yml a partir do terraform output
    └── playbooks/
        ├── 01-prepare-nodes.yml    ← swap, kernel modules, sysctl
        ├── 02-install-containerd.yml ← runtime de containers
        ├── 03-install-kubeadm.yml  ← kubeadm, kubelet, kubectl
        ├── 04-init-master.yml      ← kubeadm init + Calico CNI
        └── 05-join-workers.yml     ← kubeadm join nos workers
```

## Fluxo de execução completo

```
1. Preencher terraform-proxmox/terraform.tfvars com valores reais
2. cd terraform-proxmox && terraform init && terraform plan && terraform apply
3. cd ../ansible-k8s && bash inventory/generate_inventory.sh
4. ansible-playbook -i inventory/hosts.yml site.yml
```

## Regras deste projeto

- **Terraform**: nunca execute `apply` sem revisar o `plan` primeiro
- **Secrets**: credenciais do Proxmox via variáveis de ambiente (`source ~/.env.proxmox`), nunca em `.tfvars` versionados
- **Cloud-init**: IPs e chaves SSH são injetados exclusivamente via cloud-init — não altere manualmente as VMs
- **Inventário Ansible**: sempre regere com `generate_inventory.sh` após `terraform apply`
- **Ansible**: execute `ansible-lint --profile production` antes de qualquer playbook em produção
- **Kubernetes**: versão `1.29` com containerd como CRI e Calico como CNI

## Pré-requisitos

```bash
# Ferramentas necessárias
terraform >= 1.6.0
ansible >= 2.14
ansible-lint >= 6.0
python3 >= 3.10
jq >= 1.6

# Variáveis de ambiente (carregar antes de qualquer operação)
source ~/.env.proxmox

# Chave SSH para as VMs
ssh-keygen -t ed25519 -f ~/.ssh/id_ed25519_proxmox -C "lab-k8s-proxmox"
```
